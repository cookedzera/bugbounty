## Generic.Money (GUSD) ‚Äî DEEP HUNT (2026-06-13) ‚Äî VERDICT: CLEAN core / RESIDUAL dependencies
Repos: `generic-money/generic-protocol` (core, ~4.8k LoC) + `MetaFi-Labs/generic-bridging` (omnichain, ~2.1k LoC). Fully bespoke ERC-7575 stablecoin. NO push-button fund-critical bug found. Notably well-engineered & defensively coded despite being unaudited.
Reviewed in depth:
- **Decimal conversion (dev's #1 worry):** CLEAN. `_upscale`/`_downscale` with explicit rounding; ALL rounding favors protocol ‚Äî mint cost Ceil, redeem payout Floor, deposit shares Floor. `mint()` Ceil prevents free shares (non-zero asset cost enforced). No inflation attack: shares are fixed 1:1 with normalized assets (no proportional ERC4626 share price to manipulate).
- **Peg / pricing:** CLEAN. Deposit credits asset at min(oracle,$1) & fixed $1 share price; redemption charges asset at max(oracle,$1) & dynamic share price min($1, backing/supply). Robustly blocks depeg arbitrage (verified algebraically both directions ‚Äî extracting cheap OR expensive asset is ‚â§ break-even). 
- **Atomic round-trip (A2):** CLEAN. `secondaryActionFee` penalizes same-tx deposit+withdraw; flags `_depositedInTx`/`_withdrawnInTx` are native 0.8.29 `transient` (auto-reset per tx). Cross-tx stable-swap is value-neutral (by design).
- **Oracle (PriceFeedManager):** CLEAN ‚Äî the Vena lesson done RIGHT: reads feed `decimals()` dynamically & normalizes `answer*10^(18-dec)`, checks answer>0, staleness (heartbeat+buffer), decimals‚â§18.
- **Mint authority:** CLEAN. GUSD = GenericUnit wrapped 1:1 (lock-and-mint, verified). GenericUnit.mint is `onlyOwner` (controller). Public `burn(addr,amt)` only burns own/allowance (gifts value to others, not exploitable).
- **Extraction paths:** CLEAN, all role-gated. `sellRewards`/`claimRewards` can't touch vault collateral (`Reward_SameAssets`), share-supply-unchanged + exact-amount + slippage checks. `distributeYield` only mints surplus buffer (value-totalSupply) when fully backed.
- **Cross-chain mint (bridging):** CLEAN auth. Inbound `settleInboundMessage` requires msg.sender=registered local adapter AND remoteSender=registered remote adapter; LayerZero OApp adds endpoint+peer check. L2 mint backed by L1 lock/escrow (BridgeCoordinatorL1 escrow balance sanity-checked). Replay handled by LZ exactly-once + try/catch‚ÜífailedMessageExecutions‚Üírollback (no retry-then-rollback double-spend; failed msg never mints).
- **Predeposit lifecycle:** CLEAN. State machine ENABLED‚Üí{DISPATCHED xor WITHDRAWN} mutually exclusive (both transitions require ENABLED, no DISPATCHED‚ÜîWITHDRAWN), entries deleted on consume ‚Üí no double bridge+withdraw.
RESIDUAL / watch (honest, not bugs):
1. **External strategy trust** ‚Äî backing/redemption price depends on external ERC4626 (Morpho/Aave/Sky) `convertToAssets`. A lossy/manipulable strategy impairs redemption. Donation-inflation NOT profitable (benefit is pro-rata to all holders > attacker cost). RESIDUAL dependency.
2. **Privileged roles** ‚Äî PRICE_FEED_MANAGER can set any feed (powerful misprice lever); managers/admin govern. Centralization, not a code bug.
3. **No coordinator-level messageId replay guard** ‚Äî relies on LayerZero/Linea canonical exactly-once. If a future non-canonical adapter without dedup is registered, replay-mint risk. Architectural note.
Tooling note: Foundry binary download blocked (GitHub releases reset) ‚Üí did full static + pure-arithmetic analysis, no runnable PoC. Verdict honestly CLEAN (proof-of-clean), not a fabricated bug.
